## 1. 점검 범위

## 3. 상세 분석

### F-01. 러너 대시보드 — 최근 피드백 링크 없음 (Critical)

**현상**
`RecentFeedbackList` 컴포넌트(`src/features/learner-dashboard/components/recent-feedback-list.tsx`)의 피드백 카드가 순수 `<Card>`로 렌더링되어, 클릭이나 링크가 전혀 없다.
학습자가 대시보드에서 "채점 완료" 또는 "재제출 요청" 피드백을 발견해도 상세 내용을 확인할 방법이 없다.

**영향**
- 학습자 핵심 플로우(피드백 확인 → 재제출) 단절
- 대시보드의 "최근 피드백" 섹션이 사실상 장식에 불과

**개선안**
1. `recentFeedbackSchema`에 `courseId` 필드 추가 (F-10과 연관)
2. 백엔드 서비스에서 `assignments.course_id`를 함께 조회하여 반환
3. 피드백 카드를 `<Link href={/courses/my/${courseId}/assignments/${assignmentId}/feedback}>` 로 감싸기

**예상 공수**: 2~3h (스키마 변경 + 백엔드 수정 + 프론트 수정)

---

### F-02. 강사 코스 수정 → 과제 생성 진입점 없음 (Critical)

**현상**
`CourseFormPage`(`src/features/course-management/components/course-form-page.tsx`)에서 코스를 수정한 뒤, 해당 코스에 과제를 생성하려면 URL을 직접 입력하거나 대시보드로 돌아가야 한다. "과제 만들기" 버튼이 코스 수정 페이지 어디에도 없다.

또한 `instructor/assignments/new` 페이지에서 코스를 드롭다운으로 선택해야 하므로, 어느 코스의 과제인지 컨텍스트가 이미 있음에도 불구하고 강사가 다시 선택해야 한다.

**영향**
- 코스 생성 → 과제 생성의 핵심 Instructor 여정이 끊김
- 기본 CRUD 플로우에서 불필요한 단계 추가

**개선안**
1. 코스 수정 페이지 하단에 "이 코스에 과제 만들기" 버튼 추가
2. 링크에 `?courseId=xxx` 쿼리 파라미터 포함하여 과제 생성 폼에서 코스 자동 선택

**예상 공수**: 1~2h

---

### F-03. Learner 헤더에 코스 카탈로그 링크 없음 (High)

**현상**
`AppHeader`(`src/components/app-header.tsx`)의 `ROLE_NAV_CONFIG.learner`에 "내 코스"(`/courses/my`) 하나만 존재한다. 학습자가 새 코스를 탐색하려면 홈(`/`)으로 돌아가야 하는데, 헤더의 LMS 로고가 홈 링크라는 것을 인지하기 어렵다.

**영향**
- 코스 탐색 → 수강신청 흐름의 진입점이 불분명
- 신규 코스 발견 빈도 저하

**개선안**
- `ROLE_NAV_CONFIG.learner`에 `{ href: "/", label: "코스 탐색", icon: Search }` 항목 추가

**예상 공수**: 15min

---

### F-04. 코스 상세 — 수강신청 후 안내 없음 (High)

**현상**
`EnrollButton`(`src/features/course/components/enroll-button.tsx`)에서 수강신청 성공 후 페이지가 그대로 유지된다. 학습자가 "이제 어디로 가야 하지?"라는 막막함을 느낄 수 있다. 과제 목록이나 대시보드로의 유도가 없다.

**영향**
- 수강신청 후 다음 행동(과제 확인, 대시보드 확인)까지 도달률 하락

**개선안**
- 수강신청 성공 후 toast 알림에 "내 학습에서 확인하기" 링크 포함
- 또는 수강신청 버튼을 "내 학습으로 이동" 버튼으로 전환

**예상 공수**: 1h

---

### F-05. 수강 중 코스 카드에서 성적 보기 진입점 없음 (High)

**현상**
`EnrolledCourseList`(`src/features/learner-dashboard/components/enrolled-course-list.tsx`)의 코스 카드가 `<Link href={/courses/${course.id}}>` 즉 일반 코스 상세 페이지로 연결된다. 수강 중인 학습자 입장에서는 과제 목록이나 성적 페이지(`/courses/my/${id}/grades`)로 직접 가고 싶을 수 있지만 진입점이 없다.

현재 성적 페이지는 어디에서도 링크되지 않는 고립된 페이지다(피드백 상세의 "성적으로" 뒤로가기만 존재).

**영향**
- 성적 확인 기능의 접근성 극히 낮음
- UC-006(성적/피드백 열람) 기능이 사실상 발견 불가

**개선안**
- 수강 중 코스 카드에 "성적 보기" 아이콘 버튼 추가
- 또는 코스 카드 클릭 시 학습자용 코스 허브 페이지(과제 목록 + 성적 + 진행률)로 이동

**예상 공수**: 2~3h

---

### F-06. 과제 상세 페이지 — 코스 컨텍스트 표시 없음 (Medium)

**현상**
`AssignmentDetailPage`(`src/features/assignment-detail/components/assignment-detail-page.tsx`)에서 뒤로가기 버튼이 `/courses/my`(내 학습)로 바로 이동한다.

현재 과제가 어느 코스에 속하는지 제목 영역에 표시되지 않으며, 해당 코스의 다른 과제나 성적으로 이동할 수 없다.

**영향**
- 같은 코스의 여러 과제를 순차적으로 확인하기 불편
- 코스 맥락 없이 과제만 보게 되어 혼란

**개선안**
- 과제 제목 위에 코스명을 부제로 표시하고, 코스 상세로의 링크 추가
- 뒤로가기를 `/courses/my` 대신 코스 컨텍스트 페이지로 변경 검토

**예상 공수**: 1h

---

### F-07. 강사 제출물 관리 — 뒤로가기 없음 (Medium)

**현상**
`SubmissionListPage`(`src/features/assignment-management/components/submission-list-page.tsx`)에 상위 페이지로 돌아가는 네비게이션이 전혀 없다. 강사가 채점 완료 후 대시보드로 돌아가려면 브라우저 뒤로가기를 사용하거나 헤더의 "강사 대시보드"를 클릭해야 한다.

**영향**
- 채점 워크플로우 완료 후 다음 행동까지 마찰

**개선안**
- 페이지 상단에 "← 대시보드" 또는 "← 코스명" 뒤로가기 버튼 추가

**예상 공수**: 30min

---

### F-08. 강사 코스 수정 — 뒤로가기 없음 (Medium)

**현상**
`CourseFormPage`에 대시보드로 돌아가는 버튼이 없다. 코스 수정 후 사용자가 어디로 가야 하는지 불분명하다.

**개선안**
- 페이지 상단에 "← 대시보드" 링크 추가

**예상 공수**: 15min

---

### F-09. 강사 과제 생성 — 뒤로가기 없음 (Medium)

**현상**
`AssignmentFormPage`(create 모드)에 상위 페이지로 돌아가는 네비게이션이 없다.

**개선안**
- 페이지 상단에 "← 대시보드" 링크 추가

**예상 공수**: 15min

---

### F-10. recentFeedback 스키마에 courseId 누락 (Medium)

**현상**
`recentFeedbackSchema`(`src/features/learner-dashboard/backend/schema.ts`)에 `courseId`가 포함되어 있지 않다.
따라서 프론트에서 피드백 카드에 상세 링크(`/courses/my/{courseId}/assignments/{assignmentId}/feedback`)를 구성할 수 없다.

**영향**
- F-01 해결의 전제조건

**개선안**
1. 스키마에 `courseId: z.string().uuid()` 추가
2. 서비스에서 `assignments.course_id`를 함께 select 하여 매핑

**예상 공수**: 1h (F-01과 병합 처리 가능)

---

### F-11. 강사 코스 카드 — 클릭 시 바로 수정 페이지 (Low)

**현상**
`MyCourseList`(`src/features/instructor-dashboard/components/my-course-list.tsx`)에서 코스 카드 클릭 시 `Link href={/instructor/courses/${course.id}/edit}`으로 바로 수정 페이지에 진입한다. 코스 개요를 먼저 확인하고 싶은 경우 불편하다.

**영향**
- 강사가 코스 상태를 빠르게 조회하기 어려움 (수정 폼이 바로 로드됨)

**개선안**
- 카드 클릭 → 코스 상세(읽기 전용) + 상단에 "수정" 버튼 패턴 검토
- 또는 현 상태 유지하되, 드롭다운의 "수정" 메뉴와 역할 분리

**예상 공수**: 3~4h (코스 상세 읽기 전용 페이지 추가 시)

---

### F-12. 피드백 상세 → 과제 상세 링크 없음 (Low)

**현상**
`FeedbackPage`(`src/features/grades/components/feedback-page.tsx`)의 뒤로가기가 성적 페이지(`/courses/my/${courseId}/grades`)로만 연결된다. "재제출 요청" 상태에서 학습자가 바로 과제 제출 페이지로 이동하고 싶어도 직접 경로를 찾아야 한다.

**개선안**
- 재제출 요청 상태일 때 "과제로 돌아가기" 또는 "재제출하기" 버튼 추가

**예상 공수**: 30min

## 6. 권장 우선순위

1. **F-01 + F-10** — 피드백 링크 (최우선. 대시보드 핵심 기능 복구)
2. **F-03** — 헤더 코스 탐색 링크 (15분 작업, 효과 큼)
3. **F-02** — 코스 → 과제 생성 (강사 핵심 플로우)
4. **F-05** — 성적 진입점 (학습자 핵심 기능 접근성)
5. **F-07, F-08, F-09** — 강사 뒤로가기 일괄 추가
6. **F-04** — 수강신청 후 안내
7. **F-06** — 과제 상세 컨텍스트
8. **F-12** — 피드백 → 재제출 링크
9. **F-11** — 강사 코스 카드 (설계 변경 필요, 후순위)
